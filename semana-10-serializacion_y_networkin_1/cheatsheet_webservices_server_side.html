Cheat Sheet: Web Services Server Side (Flask)

Guia rapida para construir APIs server side en Python con Flask. Incluye rutas, metodos HTTP, parsing de request, respuestas JSON, manejo de errores, validaciones, seguridad base y pruebas.

Stack: Python + Flask + requests
Formato comun: JSON
Base local: http://localhost:4444
Regla: endpoints pequenos y claros
Boot

1. Estructura minima

from flask import Flask

app = Flask(__name__)

@app.get("/")
def health():
    return {"status": "ok"}

if __name__ == "__main__":
    app.run(host="localhost", port=4444)

Route

2B. Ejemplos de app.route()

from flask import request

# 1) Forma clasica con metodos explicitos
@app.route("/ping", methods=["GET"])
def ping():
    return {"msg": "pong"}

# 2) Mismo endpoint con GET y POST
@app.route("/notes", methods=["GET", "POST"])
def notes():
    if request.method == "GET":
        return {"data": []}
    payload = request.get_json(force=True)
    return {"created": payload}, 201

# 3) Parametro dinamico y conversor de tipo
@app.route("/users/<int:user_id>", methods=["GET"])
def get_user(user_id):
    return {"id": user_id}

# 4) Segmento tipo path (incluye /)
@app.route("/files/<path:file_path>", methods=["GET"])
def file_info(file_path):
    return {"path": file_path}

# 5) Dos rutas para la misma funcion
@app.route("/about")
@app.route("/acerca")
def about():
    return {"service": "server-side api"}
  • Usa methods para controlar que verbos acepta cada ruta
  • Conversores utiles: string, int, float, path, uuid
Input

3. Leer parametros y body

from flask import request

@app.get("/search")
def search():
    q = request.args.get("q", default="", type=str)
    limit = request.args.get("limit", default=10, type=int)
    return {"q": q, "limit": limit}

@app.post("/sum")
def do_sum():
    payload = request.get_json(force=True)
    a = payload["a"]
    b = payload["b"]
    return {"result": a + b}

4. Respuestas y status code

from flask import Response
import json

@app.post("/users")

@app.get("/plain")
def plain_text():
    return Response(
        response="pong",
        status=200,
        content_type="text/plain"
    )

@app.get("/custom")
def custom_json():
    body = json.dumps({"msg": "accepted"})
    return Response(body, status=202, content_type="application/json")

Errors

5. Manejo de errores

from flask import abort

@app.get("/users/<int:user_id>")
def get_user(user_id):
    user = DB.get(user_id)
    if user is None:
        abort(404)
    return user

@app.errorhandler(400)
def bad_request(_):
    return {"error": "bad request"}, 400

@app.errorhandler(404)
def not_found(_):
    return {"error": "not found"}, 404
  • No ocultes errores con status 200
  • Devuelve mensajes claros y consistentes
Validation

6. Validacion basica

@app.post("/products")
def create_product():
    data = request.get_json(force=True)

    required = ["name", "price"]
    missing = [k for k in required if k not in data]
    if missing:
        return {"error": "missing fields", "fields": missing}, 400

    if not isinstance(data["price"], (int, float)):
        return {"error": "price must be numeric"}, 400

    return {"ok": True}, 201
Security

7. Seguridad minima

  • Jamás hardcodear claves en el codigo
  • Usar variables de entorno para secretos
  • Validar entradas siempre
  • Limitar payload size y aplicar timeout
  • No exponer stacktrace en produccion
  • Usar HTTPS fuera de local

Testing

8. Probar desde cliente (requests)

import requests

base = "http://localhost:4444"

r = requests.get(f"{base}/search", params={"q": "cat", "limit": 2})
print(r.status_code, r.json())

body = {"a": 10, "b": 5}
r = requests.post(f"{base}/sum", json=body)
print(r.status_code, r.json())
