🏗️ Información General

General

¿Qué es esta API?

API pública del Coordinador Eléctrico Nacional de Chile. Proporciona datos en tiempo real y históricos del sistema eléctrico (generación, demanda, costos, topología).

Detalle

Servicios disponibles:

  • Operaciones — generación, desviación, CNE, potencia neta, tramos, nodos
  • Información Pública (SIP) — datos del sistema interconectado
  • Mercados — pronósticos de demanda, ERV, centrales de pasada
  • Medidas (PRTME) — mediciones del sistema
  • Planificación — programación de operación
General

URL Base y estructura

Todas las requests al servicio de Operaciones usan la misma URL base y se diferencian por endpoint.

Estructura 
# URL Base (Operaciones)
BASE = "https://operacion.api.coordinador.cl"

# Estructura general:
# BASE + /servicio/version/recurso

# Ejemplos:
"/reportes/v3/generation"
"/reportes/v3/deviation"
"/reportes/v3/cne"
"/net-power/v1/"
"/tramos/v1"
"/proyectos/v1"
General

Registro y obtención de API Key

Para usar la API necesitas registrarte en el portal y obtener una clave de usuario (user_key).

Pasos

Cómo obtener acceso:

  • 1. Ir a portal.api.coordinador.cl
  • 2. Registrarse (crear cuenta)
  • 3. Suscribirse a un plan (hay planes gratuitos)
  • 4. Obtener tu user_key desde tu perfil

⚠️ La key se pasa como parámetro de consulta, NO en headers.

🔐 Autenticación

Auth

Método de autenticación: user_key

A diferencia de muchas APIs, aquí la clave va como query parameter, NO en headers tipo Bearer.

Código 
API_KEY = "tu_api_key_aqui"

# ✅ CORRECTO: user_key como parámetro
parametros = {
    "user_key": API_KEY,
    "date": "2025-06-15"
}
resp = requests.get(BASE + endpoint, params=parametros)

# ❌ INCORRECTO: NO usar headers
# headers = {"Authorization": f"Bearer {API_KEY}"}
# headers = {"Ocp-Apim-Subscription-Key": API_KEY}
Gotcha

Errores comunes de autenticación

Si recibes 401 o 403, probablemente estás pasando la key de forma incorrecta.

Errores

Problemas frecuentes:

  • 401 Unauthorized → API key inválida o expirada
  • 403 Forbidden → No tienes suscripción al plan
  • "Exceeded maximum number of pages" → No usar page/pageSize manualmente
  • Key en header → Mover a params
# Verificar error:
if resp.status_code != 200:
    print(resp.status_code, resp.text[:300])

📊 Endpoints de Reportes (los más usados)

Endpoint

GET /reportes/v3/generation

Clasificación de generación por fuente energética (hidráulica, térmica, eólica, solar, BESS, geotérmica).

Código 
import requests

BASE = "https://operacion.api.coordinador.cl"
endpoint = "/reportes/v3/generation"

params = {
    "user_key": API_KEY,
    "date": "2025-06-15"  # YYYY-MM-DD
}
resp = requests.get(BASE + endpoint, params=params)
data = resp.json()

# Respuesta: 6 categorías por día
for item in data["content"]:
    print(item["description"], item["dailyCurrent"])
Endpoint

GET /reportes/v3/deviation

Reporte de desviación desde el sistema OpReal. Muestra diferencias entre generación programada y real.

Código 
endpoint = "/reportes/v3/deviation"

params = {
    "user_key": API_KEY,
    "date": "2025-06-15"
}
resp = requests.get(BASE + endpoint, params=params)
data = resp.json()

print(f"Total: {data['totalElements']} registros")
for item in data["content"]:
    print(item)
Endpoint

GET /reportes/v3/cne

Reporte CNE (Comisión Nacional de Energía) desde el sistema OpReal.

Código 
endpoint = "/reportes/v3/cne"

params = {
    "user_key": API_KEY,
    "date": "2025-06-15"
}
resp = requests.get(BASE + endpoint, params=params)
data = resp.json()

# Estructura similar a generation
print(data.keys())
print(data["content"])

📦 Estructura de Respuesta

Response

Formato de respuesta paginada

Las respuestas siguen un formato estándar con paginación y metadatos.

Estructura JSON 
{
  "page": 0,
  "pageSize": 20,
  "totalElements": 6,
  "totalPages": 1,
  "content": [
    {
      "description": "Hidráulica",
      "dailyCurrent": 115.053,
      "monthlyCurrentTodate": 1785.6,
      "annualCurrentTodate": 19141.0,
      "date": "2025-06-15"
    }, ...
  ]
}
Response

Campos de /generation

Cada registro del endpoint generation contiene datos diarios, mensuales y anuales en GWh.

Campos

Campos por registro:

  • description → Tipo de fuente (Hidráulica, Térmica, Eólica, Solar, BESS, Geotérmica)
  • dailyCurrent → Generación del día [GWh]
  • monthlyCurrentTodate → Acumulado del mes [GWh]
  • annualCurrentTodate → Acumulado del año [GWh]
  • date → Fecha consultada (YYYY-MM-DD)

6 registros por día (uno por fuente energética)

🔄 Patrones de Uso

Patrón

Consulta básica completa

Template mínimo para consultar cualquier endpoint de la API de Operaciones.

Template 
import requests

BASE = "https://operacion.api.coordinador.cl"
API_KEY = "tu_key"

endpoint = "/reportes/v3/generation"
params = {
    "user_key": API_KEY,
    "date": "2025-06-15"
}

resp = requests.get(BASE + endpoint, params=params)

if resp.status_code == 200:
    data = resp.json()
    for item in data["content"]:
        print(item)
else:
    print(f"Error: {resp.status_code}")
Patrón

Iterar por rango de fechas → CSV

Consultar múltiples días y exportar todos los resultados a un archivo CSV.

Código 
import requests, csv

all_records = []
for day in range(1, 31):
    date_str = f"2025-06-{day:02d}"
    params = {"user_key": API_KEY, "date": date_str}
    resp = requests.get(BASE + endpoint, params=params)
    if resp.status_code == 200:
        all_records.extend(resp.json()["content"])
    else:
        print(f"Error {date_str}: {resp.status_code}")

with open("output.csv", "w", newline="") as f:
    w = csv.DictWriter(f, fieldnames=["date",
        "description", "dailyCurrent"])
    w.writeheader()
    w.writerows(all_records)
Patrón

Consulta con ID de recurso

Algunos endpoints requieren un ID en la ruta (topología, tramos, proyectos).

Código 
# Endpoint con ID en la ruta
proyecto_id = 123
endpoint = f"/proyectos/v1/{proyecto_id}"

# Topología con infotecnica_id
infotecnica_id = 456
endpoint = f"/topologia/v3/energygen/{infotecnica_id}"

resp = requests.get(BASE + endpoint, params=params)

💡 Tips y Buenas Prácticas

Tip

No usar paginación manual

A diferencia de otras APIs, incluir page/pageSize puede causar errores en esta API.

Detalle 
# ❌ EVITAR: causa error
params = {
    "user_key": API_KEY,
    "date": "2025-06-15",
    "page": 0,         # ← NO
    "pageSize": 100   # ← NO
}
# Error: "Exceeded maximum number of pages"

# ✅ CORRECTO: solo user_key + date
params = {
    "user_key": API_KEY,
    "date": "2025-06-15"
}
Tip

Formato de fecha: YYYY-MM-DD

El parámetro date siempre debe ser una fecha en formato ISO: año-mes-día.

Ejemplos 
# ✅ Formatos correctos
"2025-06-15"
"2025-01-01"
f"2025-06-{day:02d}"  # con f-string

# ❌ Formatos incorrectos
"15-06-2025"   # DD-MM-YYYY
"06/15/2025"   # MM/DD/YYYY
"2025-6-5"     # sin zero-padding

# El parámetro se llama "date", NO:
# "fecha", "fecha_inicio", "fecha_fin"
Tip

Rate limiting y cortesía

Cuando hagas múltiples consultas, considera agregar pausas para no saturar el servidor.

Código 
import time

for day in range(1, 31):
    date_str = f"2025-06-{day:02d}"
    params = {"user_key": API_KEY, "date": date_str}
    resp = requests.get(BASE + endpoint, params=params)

    if resp.status_code == 429:
        print("Rate limited! Esperando...")
        time.sleep(60)
        continue

    # Pausa de cortesía entre requests
    time.sleep(0.5)

El código 429 indica "Too Many Requests".
