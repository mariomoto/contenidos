📚 Conceptos Fundamentales

Concepto

¿Qué es un Web Service?

Un servicio accesible a través de la red que permite la comunicación entre aplicaciones usando protocolos estándar (HTTP/HTTPS).

Detalle

Componentes clave:

• Cliente: envía requests (tu código Python)
• Servidor: procesa y responde
• Protocolo: HTTP/HTTPS
• Formato: generalmente JSON o XML

El cliente NO necesita saber cómo está implementado el servidor internamente.

Concepto

URL Base + Endpoint

Toda API tiene una URL base (prefijo) y endpoints (rutas) que definen a qué recurso accedes.

Ejemplo 
# URL Base: prefijo común a todas las rutas
BASE = "https://dummyjson.com/"

# Endpoints: rutas específicas
endpoint_recetas = "recipes/"
endpoint_productos = "products/"

# URL final = BASE + endpoint
url = BASE + endpoint_recetas
# → "https://dummyjson.com/recipes/"
Concepto

Métodos HTTP

Definen la acción que quieres realizar sobre un recurso del servidor.

Resumen

GET → Obtener/consultar datos
POST → Crear un recurso nuevo
PUT → Reemplazar un recurso completo
PATCH → Modificar parcialmente
DELETE → Eliminar un recurso

requests.get(url)
requests.post(url, data=body)
requests.put(url, data=body)
requests.delete(url)
Concepto

Status Codes

Códigos numéricos que indican el resultado de la petición HTTP.

Códigos comunes

2xx – Éxito:
200 OK · 201 Created · 204 No Content

4xx – Error del cliente:
400 Bad Request · 401 Unauthorized
403 Forbidden · 404 Not Found

5xx – Error del servidor:
500 Internal Server Error

if resp.status_code == 200:
    data = resp.json()

🐍 Librería requests

Básico

Instalación e importación

La librería requests es externa y debe instalarse con pip.

Código 
# Instalación (en terminal)
python3 -m pip install requests

# Importación
import requests

Si python3 no funciona, prueba con python, py o py3.

GET

Consulta GET básica

Usa requests.get(url) para obtener datos de un servidor. Retorna un objeto Response.

Código 
import requests

url = "https://api.github.com/repos/IIC2233/Syllabus/contents/Actividades"
respuesta = requests.get(url)

# Verificar estado
print(respuesta.status_code)  # 200

# Obtener datos como dict/list
data = respuesta.json()
GET

Parámetros de consulta (params)

Dos formas de pasar parámetros: en la URL directamente o con el argumento params (recomendado).

Código 
# ❌ En la URL (menos legible)
url = "https://api.com/search?q=python&limit=5"
resp = requests.get(url)

# ✅ Con params (recomendado)
parametros = {
    "q": "python",
    "limit": 5
}
resp = requests.get(
    "https://api.com/search",
    params=parametros
)
POST

Crear recursos con POST

Envía datos al servidor para crear un nuevo recurso. Usa el parámetro data o json.

Código 
data = {
    "title": "Mi producto",
    "price": 2233,
}

# Opción 1: data (form-encoded)
resp = requests.post(url, data=data)

# Opción 2: json (Content-Type: application/json)
resp = requests.post(url, json=data)

print(resp.status_code)  # 201 Created
print(resp.json())       # recurso creado

🔐 Autenticación

Auth

Autenticación con Headers

Los headers permiten enviar credenciales (tokens) para verificar tu identidad ante la API.

Código 
# Obtener token
body = {"username": "user", "password": "pass"}
resp = requests.post(login_url, data=body)
token = resp.json()["accessToken"]

# Usar token en headers
headers = {
    "Authorization": f"Bearer {token}"
}
resp = requests.get(url, headers=headers)
Auth

API Key como parámetro

Algunas APIs usan una clave directamente en los parámetros de la URL en lugar de en headers.

Código 
# API Key en params (ej: OMDb, Coordinador)
parametros = {
    "apikey": "tu_api_key",
    "t": "Inception"
}
resp = requests.get(BASE, params=parametros)

# Otra variante
parametros = {
    "user_key": API_KEY,
    "date": "2025-06-15"
}
resp = requests.get(url, params=parametros)

📦 Manejo de Respuestas

Response

Atributos del objeto Response

El objeto Response contiene toda la información de la respuesta HTTP.

Referencia 
resp.status_code   # int: 200, 404...
resp.reason        # str: "OK", "Not Found"
resp.json()        # dict/list (si es JSON)
resp.text          # str: body como texto
resp.content       # bytes: body crudo
resp.headers       # dict: headers respuesta
resp.url           # str: URL final usada
resp.encoding      # str: encoding detectado
Response

Contenido no-JSON (imágenes, etc.)

No todas las APIs devuelven JSON. Verifica el Content-Type antes de parsear.

Código 
from json import JSONDecodeError

try:
    data = resp.json()
except JSONDecodeError:
    content_type = resp.headers["Content-Type"]
    print(f"No es JSON: {content_type}")
    # Para imágenes, guardar bytes:
    with open("img.png", "wb") as f:
        f.write(resp.content)

🔄 Patrones Comunes

Patrón

Búsqueda → Detalle

Patrón típico: primero buscar (lista), luego consultar detalle de cada resultado.

Código 
# 1. Buscar (retorna lista resumida)
params = {"apikey": KEY, "s": "Matrix"}
results = requests.get(BASE, params=params).json()

# 2. Detalle de cada resultado
for item in results["Search"]:
    detail_params = {
        "apikey": KEY,
        "i": item["imdbID"]
    }
    detail = requests.get(
        BASE, params=detail_params
    ).json()
    print(detail["Plot"])
Patrón

Iteración por fechas → CSV

Consultar una API día a día y acumular resultados para exportar a CSV.

Código 
import csv

all_records = []
for day in range(1, 31):
    date_str = f"2025-06-{day:02d}"
    params = {"user_key": KEY, "date": date_str}
    resp = requests.get(url, params=params)
    if resp.status_code == 200:
        all_records.extend(resp.json()["content"])

with open("out.csv", "w", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=[...])
    writer.writeheader()
    writer.writerows(all_records)
Patrón

Manejo de errores robusto

Siempre verifica el status code antes de procesar la respuesta.

Código 
resp = requests.get(url, params=params)

if resp.status_code == 200:
    data = resp.json()
    # Procesar data...
elif resp.status_code == 401:
    print("No autorizado. Revisa tu API key.")
elif resp.status_code == 404:
    print("Recurso no encontrado.")
else:
    print(f"Error {resp.status_code}: {resp.reason}")
    print(resp.text[:200])

⚡ Quick Reference

Resumen

Anatomía de una request

Los 4 componentes principales que puedes configurar en cada petición.

Estructura 
requests.<método>(
    url,              # BASE + endpoint
    params={...},     # query string (?key=val)
    data={...},       # body (form-encoded)
    json={...},       # body (JSON)
    headers={...},    # auth, content-type
)

params → se añaden a la URL
data/json → van en el body
headers → metadatos de la petición

Tip

Diferencia entre data y json

¿Cuándo usar data= vs json= en POST?

Comparación 
# data= → Content-Type: form-urlencoded
# Útil para formularios y APIs simples
requests.post(url, data={"key": "val"})

# json= → Content-Type: application/json
# Útil para APIs modernas (REST)
requests.post(url, json={"key": "val"})

⚠️ Con data= si la API espera JSON, debes hacer json.dumps(body) manualmente.
